**1. Create `app/api/admin/karute/[id]/route.ts`:**
- Export async function GET(request: Request, { params }: { params: Promise<{ id: string }> })
- Admin auth check (getAdminSession), return 401 if unauthorized
- Fetch karute record by ID with includes:
  - entries (ordered by displayOrder asc)
  - customer (select: id, name)
  - worker (select: id, name)
  - booking (select: id, startsAt)
  - recordingSession -> segments (ordered by segmentIndex asc)
- Return 404 if not found
- Return JSON with the full record data

**2. Create `src/components/karute/StatusBadge.tsx`:**
- `'use client'` component
- Props: `{ status: string; locale?: string }`
- Color mapping per user decision: DRAFT = `bg-gray-100 text-gray-700`, REVIEW = `bg-yellow-100 text-yellow-700`, APPROVED = `bg-green-100 text-green-700`
- Label mapping: DRAFT = '下書き'/'Draft', REVIEW = '確認中'/'Review', APPROVED = '承認済'/'Approved'
- Render as `<span>` with rounded-full pill styling

**3. Create `src/components/karute/ConfidenceBadge.tsx`:**
- Props: `{ confidence: number }`
- If confidence < 0.7: show warning icon (SVG triangle-exclamation) with amber/orange styling and "低信頼度" tooltip
- If confidence >= 0.7: show confidence as percentage in muted text
- Use Tailwind classes only (no component library)

**4. Create `src/components/karute/EntryCard.tsx`:**
- `'use client'` component
- Props: `{ entry: KaruteEntryData; onHover: (segmentIndices: number[]) => void; onLeave: () => void; onUpdate: () => void }`
- Display as a card with:
  - Category badge (colored by category type using a lookup object, e.g., SYMPTOM=red, TREATMENT=blue, BODY_AREA=purple, etc.)
  - Content text (the classified content)
  - ConfidenceBadge component
  - Original quote in a muted/italic blockquote style
  - Tags displayed as small pills (if any)
- Inline editing: click content text to toggle edit mode
  - In edit mode: show textarea for content, Select dropdown for category
  - Save button calls updateKaruteEntryAction server action, then onUpdate (SWR mutate)
  - Cancel button reverts to display mode
- Delete button (small, muted) calls deleteKaruteEntryAction, then onUpdate
- On hover: call onHover(entry.segmentIndices) to trigger transcript highlighting
- On mouse leave: call onLeave()
- Important: Track editing state locally. When editing, do NOT let SWR revalidation overwrite the edit (the parent KaruteEditor should use `revalidateOnFocus: false`)

**5. Create `src/components/karute/EntryForm.tsx`:**
- `'use client'` component for manually adding new entries
- Props: `{ recordId: string; onAdd: () => void }`
- Form with: category Select dropdown (all KaruteEntryCategory values), content Textarea, optional original quote input
- Submit calls createKaruteEntryAction server action
- After success, clear form and call onAdd (SWR mutate)
- Collapsible -- starts as a "+ Add Entry" button, expands to show form

**6. Create `src/components/karute/TranscriptPanel.tsx`:**
- `'use client'` component
- Props: `{ segments: TranscriptionSegmentData[]; highlightedIndices: number[] }`
- Render each segment with:
  - Speaker label badge
  - Content text
  - Each segment has `id={`segment-${segment.segmentIndex}`}` for scroll targeting
- Highlighted segments get a visible background color (e.g., `bg-yellow-100`)
- When highlightedIndices changes, scroll the first highlighted segment into view using `scrollIntoView({ behavior: 'smooth', block: 'center' })`
- Use useEffect with a ref to handle scroll after render

**7. Create `src/components/karute/ApprovalControls.tsx`:**
- `'use client'` component
- Props: `{ status: string; recordId: string; onUpdate: () => void }`
- Display current StatusBadge
- Show transition buttons based on current status:
  - DRAFT: "Submit for Review" button (-> REVIEW)
  - REVIEW: "Approve" button (-> APPROVED), "Back to Draft" button (-> DRAFT)
  - APPROVED: "Reopen" button (-> DRAFT) -- approved records remain always editable per user decision
- Each button calls updateKaruteStatusAction server action, then onUpdate
- Buttons use appropriate colors: review=yellow variant, approve=green variant, reopen=gray

**8. Create `src/components/karute/index.ts`:**
