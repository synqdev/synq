<task type="auto">
  <name>Task 1: Create checkout API endpoint</name>
  <files>
    app/api/membership/checkout/route.ts
  </files>
  <action>
    Create `app/api/membership/checkout/route.ts`:

    **POST handler**: Create Stripe checkout session
    - Read `customerId` from cookies (`customerId` cookie, same pattern as booking flow in `app/actions/booking.ts`)
    - If no customerId cookie, return 401 with error message prompting registration
    - Parse and validate request body with `createCheckoutSchema` (expects { planId: uuid })
    - Look up the Customer by ID to get email (for prefilling checkout)
    - Look up the MembershipPlan by planId to get stripePriceId — return 404 if plan not found or inactive
    - Call `createCheckoutSession(customerId, planId, origin)` from payment service
      - The service creates `stripe.checkout.sessions.create()` with:
        - `mode: 'subscription'`
        - `line_items: [{ price: stripePriceId, quantity: 1 }]`
        - `customer_email: customer.email` (prefills the Stripe checkout form)
        - `metadata: { internalCustomerId: customerId }` (for checkout.session.completed)
        - `subscription_data: { metadata: { internalCustomerId: customerId } }` (for subscription lifecycle events — CRITICAL: must be subscription_data.metadata, not top-level only)
        - `success_url: ${origin}/{locale}/membership/success?session_id={CHECKOUT_SESSION_ID}`
        - `cancel_url: ${origin}/{locale}/membership/cancel`
    - Return `{ url: session.url }` — client redirects to this URL
    - Apple Pay works automatically with Stripe hosted checkout (no additional config needed)

    Export `const dynamic = 'force-dynamic'` to prevent caching.
  </action>
  <verify>
    - `npx tsc --noEmit` passes
    - Route exports POST handler
    - Contains `subscription_data.metadata` (not just top-level metadata)
    - Contains `mode: 'subscription'`
    - Returns URL for client-side redirect
  </verify>
  <done>POST /api/membership/checkout creates a Stripe checkout session with subscription mode, proper metadata propagation, and returns the hosted checkout URL</done>
