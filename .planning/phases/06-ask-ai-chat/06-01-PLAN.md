**1. Add Prisma models** to `prisma/schema.prisma`:

Add ChatConversation and ChatMessage models after the karute models section:

```prisma
model ChatConversation {
  id         String        @id @default(uuid())
  customerId String?
  customer   Customer?     @relation(fields: [customerId], references: [id], onDelete: Cascade)
  title      String?
  createdAt  DateTime      @default(now())
  updatedAt  DateTime      @updatedAt
  messages   ChatMessage[]

  @@index([customerId])
  @@index([createdAt])
}

model ChatMessage {
  id             String           @id @default(uuid())
  conversationId String
  conversation   ChatConversation @relation(fields: [conversationId], references: [id], onDelete: Cascade)
  role           String           // 'user' | 'assistant'
  content        String
  citations      Json?
  tokenCount     Int?
  createdAt      DateTime         @default(now())

  @@index([conversationId])
  @@index([createdAt])
}
```

Add `chatConversations ChatConversation[]` relation to the Customer model.

Run `npx prisma db push` to apply (or generate migration if preferred by existing workflow).

**2. Create Zod validation schemas** at `src/lib/validations/chat.ts`:

**3. Create chat service** at `src/lib/services/chat.service.ts`:

Follow the karute.service.ts pattern exactly (KaruteResult pattern, captureError, withRLSContext, Sentry).

Export these functions:

- `getOrCreateConversation(customerId: string | null)`: Find most recent conversation for this customer (or global if null). Auto-create new conversation if none exists or if last message was >24 hours ago. Return conversation with messages.

- `getChatHistory(conversationId: string)`: Load the last 20 messages for a conversation, ordered by createdAt asc. Return as `{ role, content }[]` for OpenAI message format.

- `buildChatContext(customerId: string | null, locale: string)`: Build the system prompt for OpenAI.
  - If customerId is provided (customer-scoped mode):
    - Fetch customer profile (name, notes, visitCount, lastVisitDate)
    - Fetch all karute records with entries (use aiSummary for all, full entries for last 3 only)
    - Fetch last 20 bookings with service and worker relations
    - Format into a Japanese-first system prompt with citation instructions
    - Include `[KR:record_id]` citation format instruction in prompt
  - If customerId is null (global mode):
    - Fetch recent karute records from last 30 days (max 50)
    - Format into a global system prompt
  - System prompt should instruct the AI to respond in the user's locale (ja or en)
  - Include context budget: truncate oldest karute data if estimated tokens exceed 8000

- `saveMessage(conversationId: string, role: 'user' | 'assistant', content: string, citations?: { karuteId: string; label: string }[], tokenCount?: number)`: Save a message to the database. Update conversation `updatedAt`. If this is the first user message, set conversation `title` to first 50 chars of the message.

- `getOpenAI()`: Lazy-instantiate OpenAI client (same pattern as referenced in RESEARCH.md -- check if OPENAI_API_KEY env var exists, throw if not, cache instance in module-level variable).

Use `ChatResult<T>` type alias (same pattern as `KaruteResult<T>`).

**1. Create POST streaming endpoint** at `app/api/admin/chat/route.ts`:

Follow the existing admin API route pattern (getAdminSession guard, try/catch, proper HTTP status codes).

- Auth guard: `getAdminSession()`, return 401 if not admin
- Parse and validate request body with `sendMessageSchema`
- Call `getOrCreateConversation(customerId)` to get/create conversation
- Save the user message via `saveMessage(conversationId, 'user', message)`
- Call `buildChatContext(customerId, locale)` to get system prompt (locale from request body or default 'ja')
- Call `getChatHistory(conversationId)` to get prior messages
- Create OpenAI streaming completion:
  ```typescript
  const openai = getOpenAI()
  const stream = await openai.chat.completions.create({
    model: 'gpt-4o',
    messages: [
      { role: 'system', content: systemPrompt },
      ...history,
      { role: 'user', content: message },
    ],
    stream: true,
    stream_options: { include_usage: true },
  })
  ```
- Convert to ReadableStream using TextEncoder:
  - For each chunk, emit `data: {"content":"..."}\n\n`
  - On final chunk with usage, emit `data: {"usage":{...}}\n\n`
  - Emit `data: [DONE]\n\n` at the end
  - After streaming completes, save the full assistant response via `saveMessage()` with accumulated content, parsed citations, and token count
  - Send `data: {"conversationId":"..."}\n\n` as first event so client knows the conversation ID
- Return Response with headers: `Content-Type: text/event-stream`, `Cache-Control: no-cache`, `Connection: keep-alive`
