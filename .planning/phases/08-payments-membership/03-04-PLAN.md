  <files>
    app/api/membership/checkout/route.ts
  </files>
  <action>
    Create `app/api/membership/checkout/route.ts`:

    **POST handler**: Create Stripe checkout session (one-time payment)
    - Read `customerId` from cookies (`customerId` cookie, same pattern as booking flow in `app/actions/booking.ts`)
    - If no customerId cookie, return 401 with error message prompting registration
    - Parse and validate request body with `createCheckoutSchema` (expects { planId: uuid })
    - Read locale from request headers or URL (e.g., `Accept-Language` header or pass locale in body)
    - Look up the Customer by ID to get email (for prefilling checkout)
    - Look up the MembershipPlan by planId to get stripePriceId — return 404 if plan not found or inactive
    - Resolve locale for redirect URLs at runtime:
      - Read `locale` from request headers, cookies, or pass as query param
      - Build success_url: `${origin}/${locale}/membership/success?session_id={CHECKOUT_SESSION_ID}`
      - Build cancel_url: `${origin}/${locale}/membership/cancel`
      - CRITICAL: Do NOT use `{locale}` as a literal placeholder — resolve it to actual value (e.g., 'ja' or 'en')
    - Call `createCheckoutSession(customerId, planId, origin, locale)` from payment service
    - Return `{ url: session.url }` — client redirects to this URL

    Export `const dynamic = 'force-dynamic'` to prevent caching.
  </action>
  <verify>
    - `npx tsc --noEmit` passes
    - Route exports POST handler
    - Contains `mode: 'payment'` (NOT 'subscription')
    - Contains `metadata: { internalCustomerId, planId }` at top level
    - Locale in redirect URLs is resolved at runtime, not a placeholder
    - Returns URL for client-side redirect
  </verify>
  <done>POST /api/membership/checkout creates a Stripe one-time payment checkout session with proper metadata and runtime-resolved locale in redirect URLs</done>
