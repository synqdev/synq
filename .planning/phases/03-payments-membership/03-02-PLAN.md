  <action>
    1. Create `src/lib/services/payment.service.ts`:

       **`handleStripeEvent(event: Stripe.Event)`**: Main event dispatcher
       - Idempotency guard: check `prisma.stripeEvent.findUnique({ where: { stripeEventId: event.id } })` — if exists, return early
       - Create StripeEvent record immediately after check
       - Switch on `event.type`:
         - `checkout.session.completed`: call `handleCheckoutComplete(session)`
         - `customer.subscription.updated`: call `handleSubscriptionChange(sub)`
         - `customer.subscription.deleted`: call `handleSubscriptionChange(sub)` with deleted flag
         - `invoice.paid`: call `handleInvoicePaid(invoice)`
         - `invoice.payment_failed`: call `handlePaymentFailed(invoice)`

       **`handleCheckoutComplete(session: Stripe.Checkout.Session)`**: Inside a `prisma.$transaction`:
       - Extract `internalCustomerId` from `session.metadata`
       - Extract `stripeCustomerId` from `session.customer` (cast to string)
       - Extract `stripeSubscriptionId` from `session.subscription` (cast to string)
       - Update Customer: set `stripeCustomerId`, `stripeSubscriptionId`, `subscriptionStatus: 'active'`
       - Find MembershipPlan by matching stripePriceId (retrieve from Stripe subscription's price)
       - Create Membership record: customerId, planId, stripeSubscriptionId, status 'active', currentPeriodStart/End from subscription
       - Create Payment record: customerId, stripeSessionId, amount (session.amount_total), currency (session.currency ?? 'jpy'), status 'succeeded'

       **`handleSubscriptionChange(sub: Stripe.Subscription)`**:
       - Find Membership by stripeSubscriptionId
       - Map Stripe status to internal status: 'active' -> 'active', 'past_due' -> 'past_due', 'canceled' -> 'canceled', 'unpaid' -> 'past_due'
       - Update Membership: status, currentPeriodStart/End, canceledAt if canceled
       - Update Customer: subscriptionStatus

       **`handleInvoicePaid(invoice: Stripe.Invoice)`**:
       - Find Customer by stripeCustomerId (from invoice.customer)
       - Create Payment: stripeInvoiceId, amount (invoice.amount_paid), currency, status 'succeeded'
       - Update Membership currentPeriodEnd if subscription invoice

       **`handlePaymentFailed(invoice: Stripe.Invoice)`**:
       - Find Customer by stripeCustomerId
       - Create Payment: stripeInvoiceId, amount (invoice.amount_due), status 'failed'
       - Update Membership status to 'past_due'
       - Update Customer subscriptionStatus to 'past_due'

       **`createCheckoutSession(customerId, planId, origin)`**: For use by checkout API
       - Look up MembershipPlan by planId to get stripePriceId
       - Look up Customer by customerId to get email
       - Call `getStripe().checkout.sessions.create()` with mode 'subscription', line_items, metadata and subscription_data.metadata containing internalCustomerId, success_url, cancel_url
       - Return session.url

       **`getPaymentsByCustomer(customerId, page, limit)`**: Paginated payment history
       - Query Payment where customerId, orderBy createdAt desc, skip/take for pagination
       - Return { payments, total }

    2. Create `src/lib/services/membership.service.ts`:

       **`getMembershipPlans(activeOnly?: boolean)`**: List plans
       - Query MembershipPlan, optionally filter isActive: true, orderBy priceJpy asc
       - Return plans array

       **`getMembershipPlanById(planId: string)`**: Single plan
       - Query by id, return plan or null

       **`getActiveMembership(customerId: string)`**: Get customer's active membership
       - Query Membership where customerId AND status in ['active', 'past_due'], include plan
       - Return membership with plan details or null

       **`getMembershipByCustomer(customerId: string)`**: All memberships for a customer
       - Query Membership where customerId, orderBy createdAt desc, include plan
       - Return array

       **`createMembershipPlan(data)`**: Admin create plan
       - Create MembershipPlan record from validated data
       - Return created plan

       **`updateMembershipPlan(id, data)`**: Admin update plan
       - Update MembershipPlan by id
       - Return updated plan

       **`deactivateMembershipPlan(id)`**: Admin soft-delete (per project convention)
       - Update isActive to false
       - Return updated plan

    Follow existing service patterns from `booking.service.ts` and `customer.service.ts` — use prisma import from `@/lib/db/client`, proper error handling, typed return values.

  <action>
    Create `app/api/stripe/webhook/route.ts`:

    1. Export `POST` handler:
       - CRITICAL: Use `await request.text()` to get raw body — NOT `request.json()`. This is required for Stripe signature verification.
       - Get `stripe-signature` header via `(await headers()).get('stripe-signature')`
       - If no signature, return 400
       - Call `getStripe().webhooks.constructEvent(body, signature, process.env.STRIPE_WEBHOOK_SECRET!)` in try/catch
       - On verification failure, log error and return 400 with "Invalid signature"
       - Call `handleStripeEvent(event)` from payment.service.ts in try/catch
       - On processing error, log error but still return 200 (to prevent Stripe retry storm for permanent errors — per research)
       - Return `NextResponse.json({ received: true })`

    2. Do NOT import `headers` from 'next/headers' if using NextRequest — use `request.headers.get('stripe-signature')` instead for consistency.

    3. This route must NOT have any authentication middleware — Stripe needs to call it directly. Signature verification IS the auth.

    4. Export `const dynamic = 'force-dynamic'` to prevent caching.
  </action>
