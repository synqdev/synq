<task type="auto">
  <name>Task 2: Create audio upload API route</name>
  <files>app/api/admin/recordings/upload/route.ts</files>
  <action>
Create `app/api/admin/recordings/upload/route.ts` following the file upload pattern from `app/api/admin/customers/[id]/intake/route.ts`.

**POST handler:**
1. Verify admin session via `getAdminSession()` — return 401 if not admin
2. Parse `request.formData()`
3. Get `file` from formData as `File | null` — return 400 if missing
4. Get `recordingSessionId` from formData as string — return 400 if missing
5. Validate MIME type against `['audio/webm', 'audio/mp4', 'audio/wav', 'audio/ogg']` — return 400 with `{ error: 'Invalid file type' }` if not allowed
6. Validate file size against `100 * 1024 * 1024` (100MB) — return 400 with `{ error: 'File too large' }` if exceeded
7. Call `uploadRecording(recordingSessionId, file)` from `@/lib/storage/recording-storage`
8. Call `updateRecordingSession({ id: recordingSessionId, audioStoragePath: result.path })` from `@/lib/services/recording.service` to persist the storage path
9. Return `NextResponse.json({ success: true, path: result.path })` with status 200

**Error handling:**
- Wrap upload/update in try/catch
- Log errors, return 500 with `{ error: 'Upload failed' }`

**No body size limit config needed** for App Router — `request.formData()` streams the body, so large files work without configuration. If issues arise, add `export const maxDuration = 60` for extended timeout.

Import `NextRequest`, `NextResponse` from `next/server`.
  </action>
  <verify>Run `npx tsc --noEmit --pretty` — file must compile without errors.</verify>
  <done>POST /api/admin/recordings/upload accepts audio files with MIME/size validation, uploads to Supabase Storage, and updates the recording session with the storage path.</done>
</task>
