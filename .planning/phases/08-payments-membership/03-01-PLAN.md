<task type="auto">
  <name>Task 2: Extend Prisma schema with payment and membership models</name>
  <files>
    prisma/schema.prisma
  </files>
  <action>
    Add the following to `prisma/schema.prisma`:

    1. Add Prisma enums for all status fields (prevents state drift from free-form strings):
       ```prisma
       enum MembershipStatus {
         active
         exhausted
       }

       enum PaymentStatus {
         succeeded
         failed
         pending
       }
       ```

    2. Add 1 new field to existing `Customer` model:
       - `stripeCustomerId  String?  @unique` — Stripe Customer ID (set after first checkout)
       Also add relation fields: `memberships Membership[]` and `payments Payment[]`
       Note: No stripeSubscriptionId or subscriptionStatus — prepaid packs are one-time payments, not subscriptions.

    3. Add `MembershipPlan` model:
       - `id` String @id @default(uuid())
       - `name` String (Japanese name, e.g. "5回プラン")
       - `nameEn` String? (English name, e.g. "5 Session Pack")
       - `description` String?
       - `priceJpy` Int (display price in yen — JPY is zero-decimal)
       - `sessionsIncluded` Int (number of sessions in this pack, e.g. 5, 10, 20)
       - `stripePriceId` String @unique (Stripe Price ID price_xxx)
       - `stripeProductId` String (Stripe Product ID prod_xxx)
       - `isActive` Boolean @default(true) (soft delete per project convention)
       - `createdAt` DateTime @default(now())
       - `updatedAt` DateTime @updatedAt
       - `memberships` Membership[] relation

    4. Add `Membership` model (represents a purchased session pack):
       - `id` String @id @default(uuid())
       - `customerId` String with relation to Customer (onDelete: Cascade)
       - `planId` String with relation to MembershipPlan
       - `sessionsRemaining` Int (starts at plan.sessionsIncluded, decremented on QR scan)
       - `status` MembershipStatus @default(active) ('active' when sessionsRemaining > 0, 'exhausted' when 0)
       - `purchasedAt` DateTime @default(now())
       - `createdAt` DateTime @default(now())
       - `updatedAt` DateTime @updatedAt
       - `@@index([customerId])`

    5. Add `Payment` model:
       - `id` String @id @default(uuid())
       - `customerId` String with relation to Customer
       - `stripeSessionId` String? @unique
       - `amount` Int (yen amount — JPY is zero-decimal, no division needed)
       - `currency` String @default("jpy")
       - `status` PaymentStatus @default(pending)
       - `description` String?
       - `createdAt` DateTime @default(now())
       - `updatedAt` DateTime @updatedAt
       - `@@index([customerId])`

    6. Add `StripeEvent` model (idempotency guard):
       - `id` String @id @default(uuid())
       - `stripeEventId` String @unique
       - `type` String (event type for debugging, e.g. 'checkout.session.completed')
       - `processedAt` DateTime? (null until processing succeeds — allows retry of failed events)

    7. Run `npx prisma migrate dev --name add-membership-payment-models` to create a reviewable migration.
    8. Run `npx prisma generate` to regenerate client types.
  </action>
  <verify>
    - `npx prisma validate` passes
    - Migration file exists in `prisma/migrations/`
    - `npx tsc --noEmit` passes with new Prisma types available
    - MembershipStatus and PaymentStatus enums exist
  </verify>
  <done>All 4 new models exist in schema with enum status fields, Customer has stripeCustomerId, Prisma migration created and applied</done>
</task>
