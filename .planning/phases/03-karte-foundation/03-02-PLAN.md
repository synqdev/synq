<task type="auto">
  <name>Task 2: Create karute service with full CRUD operations</name>
  <files>src/lib/services/karute.service.ts</files>
  <action>
Create `src/lib/services/karute.service.ts` following the Result<T> pattern from `booking.service.ts`.

**Result type:**
```typescript
export type KaruteResult<T> =
  | { success: true; data: T }
  | { success: false; error: string };
```

Note: booking.service.ts uses `{ success: true; booking: T }` — for karute, use `data` as the property name since it handles multiple entity types. Check customer.service.ts to see which property name pattern it uses, and be consistent with the more generic services.

**Functions to implement:**

1. `createKaruteRecord(input: CreateKaruteRecordInput): Promise<KaruteResult<KaruteRecordWithEntries>>`
   - Validate with createKaruteRecordSchema.safeParse
   - Create via prisma.karuteRecord.create with include: { entries: true, customer: true, worker: true, booking: true }
   - Catch errors, captureException with Sentry, return error result

2. `getKaruteRecord(id: string): Promise<KaruteResult<KaruteRecordWithEntries>>`
   - findUnique with include: { entries: true, customer: true, worker: true, booking: true, recordingSessions: true }
   - Return not found error if null

3. `getKaruteRecordsByCustomer(customerId: string): Promise<KaruteResult<KaruteRecordWithEntries[]>>`
   - findMany where customerId, orderBy createdAt desc
   - Include entries, worker, booking

4. `updateKaruteRecord(input: UpdateKaruteRecordInput): Promise<KaruteResult<KaruteRecordWithEntries>>`
   - Validate with updateKaruteRecordSchema.safeParse
   - Update via prisma.karuteRecord.update
   - Include entries, customer, worker, booking

5. `deleteKaruteRecord(id: string): Promise<KaruteResult<{ id: string }>>`
   - First, find the record with recordingSessions to get audioStoragePaths
   - Delete any associated audio files from storage (best-effort, log failures — per Pitfall 4 in RESEARCH.md)
   - Delete via prisma.karuteRecord.delete (cascades to entries)
   - Return { id }

6. `createKaruteEntry(input: CreateKaruteEntryInput): Promise<KaruteResult<KaruteEntry>>`
   - Validate, create, return

7. `updateKaruteEntry(input: UpdateKaruteEntryInput): Promise<KaruteResult<KaruteEntry>>`
   - Validate, update, return

8. `deleteKaruteEntry(id: string): Promise<KaruteResult<{ id: string }>>`
   - Delete, return { id }

**Error handling pattern (from booking.service.ts):**
- Create a `captureKaruteError(error: unknown, context: Record<string, unknown>)` helper using Sentry.captureException
- In catch blocks: call captureKaruteError, return `{ success: false, error: error instanceof Error ? error.message : 'Unknown error' }`

**Type for includes:** Define a `KaruteRecordWithEntries` type using Prisma.KaruteRecordGetPayload with the include shape, or use the return type from the Prisma query. Follow whichever pattern customer.service.ts uses.

Import validation schemas from `@/lib/validations/karute`.
Import prisma from `@/lib/db/client`.
Import Sentry from `@sentry/nextjs`.
Import deleteRecording from `@/lib/storage/recording-storage` (for cascade cleanup in deleteKaruteRecord).
