**Extend `src/lib/services/customer.service.ts`** with a new function:

```typescript
export async function getCustomerList(params: {
  search?: string;       // Searches name, email, phone (case-insensitive)
  assignedStaffId?: string;
  page?: number;         // 1-based, default 1
  pageSize?: number;     // default 25
  sortBy?: string;       // 'name' | 'visitCount' | 'lastVisitDate' | 'createdAt'
  sortOrder?: 'asc' | 'desc';
}): Promise<{ customers: CustomerListItem[]; total: number; page: number; pageSize: number }>
```

Implementation details:
- Use Prisma `where` with `OR` for search across name, email, phone using `contains` mode `insensitive`
- If assignedStaffId provided, add to `where` clause
- Use `_count` for bookings relation to compute `visitCount` (count of CONFIRMED bookings)
- Use `orderBy` with computed fields where possible, fall back to raw query for visit count sorting
- For `lastVisitDate`: aggregate max(startsAt) from bookings where status = 'CONFIRMED'
- Include pagination with `skip` and `take`
- Return the total count for pagination UI
- Include the assigned worker name (via a join or include) if assignedStaffId is set

Define a `CustomerListItem` type:
```typescript
interface CustomerListItem {
  id: string;
  name: string;
  email: string;
  phone: string | null;
  assignedStaffName: string | null;
  visitCount: number;
  lastVisitDate: string | null;  // ISO date
  nextBookingDate: string | null; // ISO date
  createdAt: string;             // ISO date
  outstandingAmount: number;
}
```
