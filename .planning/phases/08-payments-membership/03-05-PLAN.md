  <files>

  <action>
    1. Create `src/lib/services/qr.service.ts`:

       **Secret key initialization**:
       - In production (NODE_ENV === 'production'): REQUIRE QR_MEMBERSHIP_SECRET env var. Throw error if missing.
       - In development: fall back to 'dev-qr-secret' with console.warn
       - `const QR_SECRET = new TextEncoder().encode(secret)`

       **`generateMembershipQR(customerId: string): Promise<string>`**
       - Use jose `SignJWT` to create a JWT:
         - `sub`: customerId
         - `type`: 'membership'
         - Protected header: `{ alg: 'HS256' }`
         - Set issuedAt and expirationTime to '24h'
         - Sign with QR_SECRET
       - Use `qrcode` package (`QRCode.toDataURL(token, { errorCorrectionLevel: 'M', width: 300, margin: 2 })`)
       - Return the data URL string (data:image/png;base64,...)

       **`verifyMembershipQR(token: string): Promise<{ customerId: string } | null>`**
       - Use jose `jwtVerify` to verify token against same secret
       - Check `payload.type === 'membership'` and `typeof payload.sub === 'string'`
       - Return `{ customerId: payload.sub }` on success
       - Return `null` on any failure (expired, invalid, wrong type)

    2. Create `app/api/membership/qr/route.ts`:

       **GET handler**: Generate QR code for authenticated customer
       - Read `customerId` from cookies
       - If no cookie, return 401
       - Check customer has active membership with sessions remaining via `getActiveMembership(customerId)` from membership service
       - If no active membership or sessionsRemaining === 0, return 403 with message "No sessions remaining"
       - Call `generateMembershipQR(customerId)` to get data URL
       - Return `{ qrDataUrl, expiresIn: '24h', sessionsRemaining: membership.sessionsRemaining, planName: membership.plan.name }`

    3. Create `app/api/membership/verify/route.ts`:

       **POST handler**: Verify scanned QR token and redeem session (staff use)
       - Verify admin session — return 401 if not authenticated (only staff can verify)
       - Parse request body: `{ token: string }`
       - Call `verifyMembershipQR(token)` from QR service
       - If null (invalid/expired): return `{ valid: false, error: 'Invalid or expired QR code' }`
       - If valid: look up Customer by customerId, check active membership
       - If no active membership or sessionsRemaining === 0: return `{ valid: false, error: 'No sessions remaining' }`
       - Call `decrementSession(membership.id)` from membership service — decrements sessionsRemaining by 1
       - Increment `Customer.visitCount` and set `Customer.lastVisitDate` to now
       - Return `{ valid: true, customer: { name, email, sessionsRemaining: updatedMembership.sessionsRemaining, planName, sessionsIncluded: plan.sessionsIncluded } }`
  </action>

    - QR service exports generateMembershipQR and verifyMembershipQR
    - QR secret throws in production when env var missing, warns in dev
    - QR route returns data URL for authenticated customer with sessions remaining
    - Verify route checks admin session, decrements session, returns updated count
  </verify>
  <done>QR service generates signed 24h JWT tokens encoded as QR data URLs. GET /api/membership/qr returns QR for customer with sessions. POST /api/membership/verify validates token, decrements session, returns remaining count.</done>
