<task type="auto">
  <name>Task 2: Extend Prisma schema with payment and membership models</name>
  <files>
    prisma/schema.prisma
  </files>
  <action>
    Add the following to `prisma/schema.prisma`:

    1. Add 3 new fields to existing `Customer` model:
       - `stripeCustomerId  String?  @unique` — Stripe Customer ID
       - `stripeSubscriptionId String? @unique` — Active subscription ID
       - `subscriptionStatus String?` — 'active' | 'past_due' | 'canceled' | null
       Also add relation fields: `memberships Membership[]` and `payments Payment[]`

    2. Add `MembershipPlan` model:
       - `id` String @id @default(uuid())
       - `name` String (Japanese name, e.g. "月額会員")
       - `nameEn` String? (English name)
       - `description` String?
       - `priceJpy` Int (display price in yen — JPY is zero-decimal)
       - `stripePriceId` String @unique (Stripe Price ID price_xxx)
       - `stripeProductId` String (Stripe Product ID prod_xxx)
       - `intervalType` String ('month' | 'year' | 'one_time')
       - `isActive` Boolean @default(true) (soft delete per project convention)
       - `createdAt` DateTime @default(now())
       - `updatedAt` DateTime @updatedAt
       - `memberships` Membership[] relation

    3. Add `Membership` model:
       - `id` String @id @default(uuid())
       - `customerId` String with relation to Customer (onDelete: Cascade)
       - `planId` String with relation to MembershipPlan
       - `stripeSubscriptionId` String? (null for one-time purchases)
       - `status` String ('active' | 'past_due' | 'canceled' | 'expired')
       - `currentPeriodStart` DateTime?
       - `currentPeriodEnd` DateTime?
       - `canceledAt` DateTime?
       - `createdAt` DateTime @default(now())
       - `updatedAt` DateTime @updatedAt
       - `@@index([customerId])` and `@@index([stripeSubscriptionId])`

    4. Add `Payment` model:
       - `id` String @id @default(uuid())
       - `customerId` String with relation to Customer
       - `stripeSessionId` String? @unique
       - `stripeInvoiceId` String? @unique
       - `amount` Int (yen amount — JPY is zero-decimal, no division needed)
       - `currency` String @default("jpy")
       - `status` String ('succeeded' | 'failed' | 'pending')
       - `description` String?
       - `createdAt` DateTime @default(now())
       - `updatedAt` DateTime @updatedAt
       - `@@index([customerId])`

    5. Add `StripeEvent` model (idempotency guard):
       - `id` String @id @default(uuid())
       - `stripeEventId` String @unique
       - `processedAt` DateTime @default(now())

    6. Run `npx prisma db push` to apply schema changes.
    7. Run `npx prisma generate` to regenerate client types.
  </action>
  <verify>
    - `npx prisma validate` passes
    - `npx prisma db push` succeeds
    - `npx tsc --noEmit` passes with new Prisma types available
  </verify>
  <done>All 4 new models exist in schema, Customer has 3 new fields, Prisma client regenerated with new types</done>
</task>
