  <name>Task 1: Create audio utility and useAudioRecorder hook</name>
  <files>src/lib/utils/audio.ts, src/hooks/useAudioRecorder.ts</files>
  <action>
**1. Create `src/lib/utils/audio.ts`:**
- Export `getSupportedMimeType(): string` that tries MIME types in order: `audio/webm;codecs=opus`, `audio/webm`, `audio/mp4`, `audio/ogg;codecs=opus`. Uses `MediaRecorder.isTypeSupported()`. Throws Error if none supported.
- Export `AUDIO_CHUNK_INTERVAL = 1000` (ms between ondataavailable chunks).

**2. Create `src/hooks/useAudioRecorder.ts`:**
- `'use client'` directive at top.
- Export interface `UseAudioRecorderReturn`:
  - `status: 'idle' | 'recording' | 'paused' | 'stopped'`
  - `startRecording: () => Promise<void>` — calls getUserMedia, creates AudioContext + AnalyserNode (fftSize 2048), creates MediaRecorder with getSupportedMimeType(), starts recording with AUDIO_CHUNK_INTERVAL. Connects source to analyser only (NOT to destination — avoids audio feedback). Starts timer interval. Sets status to 'recording'.
  - `pauseRecording: () => void` — calls mediaRecorder.pause(), pauses timer, sets status to 'paused'.
  - `resumeRecording: () => void` — calls mediaRecorder.resume(), resumes timer, sets status to 'recording'.
  - `stopRecording: () => Promise<Blob>` — returns Promise that resolves with the audio Blob. Stops mediaRecorder, waits for onstop event, creates Blob from chunks, stops all stream tracks (releases mic), closes AudioContext, clears timer. Sets status to 'stopped'.
  - `audioBlob: Blob | null` — the recorded audio after stopping.
  - `analyserNode: AnalyserNode | null` — exposed for waveform visualization.
  - `elapsedSeconds: number` — timer counter.
  - `error: string | null` — error message if getUserMedia fails (NotAllowedError, NotFoundError).
  - `resetRecorder: () => void` — resets state back to idle, clears blob/error.

- Use refs for MediaRecorder, AudioContext, AnalyserNode, stream, chunks array, timer interval ID.
- Timer uses `setInterval` with 1-second increment (not timeDelta — per research anti-pattern).
- Cleanup on unmount: cancel any active recording, stop tracks, close AudioContext, clear timer interval.
- Handle getUserMedia errors gracefully: catch NotAllowedError (permission denied) and NotFoundError (no mic), set error state with descriptive message.
- Handle AudioContext suspended state: call `audioContext.resume()` if state is 'suspended' after creation.
  </action>
