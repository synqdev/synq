  <files>
    app/api/admin/membership-plans/route.ts
    app/api/admin/membership-plans/[id]/route.ts
  </files>
  <action>
    1. Create `app/api/admin/membership-plans/route.ts`:

       **GET handler**: List all membership plans
       - Verify admin session via `getAdminSession()` — return 401 if not authenticated
       - Accept optional query param `activeOnly` (defaults to false for admin view)
       - Call `getMembershipPlans(activeOnly)` from membership service
       - Return plans array as JSON

       **POST handler**: Create a new membership plan
       - Verify admin session
       - Parse and validate request body with `createMembershipPlanSchema` from validations
       - If Stripe product/price creation is needed (no stripePriceId provided):
         - Call `getStripe().products.create({ name, metadata: { internalPlanType: intervalType } })`
         - Call `getStripe().prices.create({ product: product.id, unit_amount: priceJpy, currency: 'jpy', recurring: intervalType !== 'one_time' ? { interval: intervalType } : undefined })`
         - Use the returned `product.id` and `price.id` as stripeProductId and stripePriceId
       - Call `createMembershipPlan(data)` with all fields including Stripe IDs
       - Return 201 with created plan

    2. Create `app/api/admin/membership-plans/[id]/route.ts`:

       **PUT handler**: Update membership plan
       - Verify admin session
       - Parse and validate with `updateMembershipPlanSchema`
       - Call `updateMembershipPlan(id, data)` — only updates display fields (name, nameEn, description, isActive). Price changes require creating a new Stripe Price (Stripe prices are immutable).
       - Return updated plan

       **DELETE handler**: Deactivate membership plan (soft delete per project convention)
       - Verify admin session
       - Call `deactivateMembershipPlan(id)`
       - Also deactivate the Stripe product: `getStripe().products.update(plan.stripeProductId, { active: false })`
       - Return 200 with deactivated plan

    Follow existing admin API patterns from `app/api/admin/customers/route.ts` — admin session check, Zod validation, try/catch with proper HTTP status codes.
  </action>

    - PUT route updates plan display fields
    - DELETE route soft-deletes plan and deactivates in Stripe
  </verify>
