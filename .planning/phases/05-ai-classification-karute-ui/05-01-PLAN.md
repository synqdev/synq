**2. Create `src/lib/services/classification.service.ts`:**
Follow the exact pattern from the RESEARCH.md classification service example:
- Lazy OpenAI client (not module-level): `let openaiClient: OpenAI | null = null; function getOpenAI()` pattern, matching Phase 4's transcription.service.ts
- Define `KaruteClassificationSchema` using Zod v4:
  ```
  z.object({
    summary: z.string(),
    entries: z.array(z.object({
      category: z.enum(['SYMPTOM', 'TREATMENT', 'BODY_AREA', 'PREFERENCE', 'LIFESTYLE', 'NEXT_VISIT', 'OTHER']),
      content: z.string(),
      confidence: z.number(),
      originalQuote: z.string(),
      segmentIndices: z.array(z.number()),
    })),
  })
  ```
- Export `type ClassificationResult = z.infer<typeof KaruteClassificationSchema>`
- Export `async function classifyAndStoreEntries(karuteRecordId: string): Promise<{ success: true; entryCount: number } | { success: false; error: string }>`
  - Fetch the karute record with its recording session's transcription segments (ordered by segmentIndex asc)
  - Return error if no segments found
  - Format transcript as: `[Segment {index}] [{speakerLabel || '?'}] {content}` lines
  - Call OpenAI gpt-4o with structured outputs:
    - System prompt in Japanese (follow RESEARCH.md example): instructs classification of treatment session transcript into categories, confidence scores, original quotes, segment indices, and session summary
    - CRITICAL: Use `z.toJSONSchema(KaruteClassificationSchema, { target: 'draft-7' })` for the response_format (NOT zodResponseFormat which is Zod v3 only)
    - Set `response_format: { type: 'json_schema', json_schema: { name: 'karute_classification', strict: true, schema: jsonSchema } }`
  - Parse response with `KaruteClassificationSchema.parse(JSON.parse(content))`
  - Store in transaction: update record summary + createMany entries with displayOrder
  - Wrap in try/catch with Sentry.captureException

**1. Create `app/api/admin/karute/[id]/classify/route.ts`:**
Using API route (not server action) because classification may take 10-60 seconds on first request (OpenAI schema caching). Follow existing API route patterns (e.g., app/api/admin/recordings/transcribe/route.ts from Phase 4):
- Export async function POST(request: Request, { params }: { params: Promise<{ id: string }> })
- Check admin auth via getAdminSession(), return 401 if unauthorized
- Extract karute record ID from params
- Call classifyAndStoreEntries(id)
- Return 200 with { success: true, entryCount } or 500 with { error }
- Add appropriate error handling
