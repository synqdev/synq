**Server Component — `app/[locale]/(admin)/admin/workers/[id]/schedule/page.tsx`:**

Follow the pattern from `app/[locale]/(admin)/admin/workers/page.tsx`:

```typescript
import { redirect, notFound } from 'next/navigation'
import { getTranslations } from 'next-intl/server'
import { prisma } from '@/lib/db/client'
import { getAdminSession } from '@/lib/auth/admin'
import { ScheduleEditor } from './schedule-editor'
```

Interface for params:
```typescript
interface PageProps {
  params: Promise<{ locale: string; id: string }>
}
```

Page logic:
1. Auth check: `getAdminSession()` → redirect to `/admin/login` if not admin
2. Extract `{ locale, id }` from `await params` (Next.js 15 async params pattern)
3. Load worker: `prisma.worker.findUnique({ where: { id } })` → call `notFound()` if null
4. Load translations: `await getTranslations('admin.schedulePage')`
5. Load existing recurring schedules: `prisma.workerSchedule.findMany({ where: { workerId: id, specificDate: null }, orderBy: { dayOfWeek: 'asc' } })`
6. Build initial schedule data for all 7 days (0-6). For each dayOfWeek, find matching record from DB results. If no record exists for a day, use defaults: `{ dayOfWeek: i, startTime: '09:00', endTime: '18:00', isAvailable: false }`.

Render:
- Back link to `/admin/workers` using the `backToWorkers` translation
- Page title using `subtitle` translation with worker name interpolation: `t('subtitle', { name: worker.name })`
- `<ScheduleEditor workerId={id} initialSchedules={schedules} />` — pass workerId and the 7-day array

**Client Component — `app/[locale]/(admin)/admin/workers/[id]/schedule/schedule-editor.tsx`:**

Mark as `'use client'`.

Import:
- `useActionState` from `react`
- `useTranslations` from `next-intl`
- `Button` from `@/components/ui`
- `Card` from `@/components/ui/card`
- `upsertWorkerSchedule` from `@/app/actions/worker-schedule`
- `generateTimeSlots` from `@/lib/utils/time`

Props interface:
```typescript
interface ScheduleEditorProps {
  workerId: string
  initialSchedules: Array<{
    dayOfWeek: number
    startTime: string
    endTime: string
    isAvailable: boolean
  }>
}
```

Component logic:
1. Generate time options: `generateTimeSlots('06:00', '23:30', 30)` — produces ['06:00', '06:30', ..., '23:00'] for select dropdowns
2. Use `useActionState` for the form action. The action should wrap `upsertWorkerSchedule`:
   ```typescript
   const [state, formAction, isPending] = useActionState(
     async (_prevState: { success: boolean; error?: string } | null, formData: FormData) => {
       return await upsertWorkerSchedule(workerId, formData)
     },
     null
   )
   ```
3. Use local React state (`useState`) to manage the 7-day schedule array so the form is interactive before submission. Initialize from `initialSchedules`.

UI structure (inside a `<form action={formAction}>`):
- A `<Card>` wrapper with the schedule title
- For each day (0-6), render a row with:
  - Day name label using `t('days.${day.dayOfWeek}')` translation
  - Availability toggle: A styled checkbox or button that toggles `isAvailable`. Use a hidden input `<input type="hidden" name={`day_${i}_isAvailable`} value={schedule[i].isAvailable ? 'true' : 'false'} />` to ensure the value is always submitted.
  - Start time `<select>` with name=`day_${i}_startTime` — populate options from the generated time slots. Disabled when `isAvailable` is false (dimmed/grayed).
  - End time `<select>` with name=`day_${i}_endTime` — same as start time. Disabled when `isAvailable` is false.
  - Visual indication: When `isAvailable` is false, show "休み"/"Day Off" badge and dim the time selectors with `opacity-50` and `pointer-events-none`.

- At the bottom: Save button using `<Button type="submit" loading={isPending}>`
  - Show `t('saving')` while pending, `t('save')` otherwise
- Status message: Show `t('saved')` in green when `state?.success === true`, show `state?.error || t('saveError')` in red when `state?.success === false`

Styling (Tailwind, matching existing admin pages):
- Grid layout for days: each row is a flex container with `items-center gap-4`
- Day label: `w-20 font-medium text-sm`
- Toggle: styled button/checkbox with green/gray states
- Selects: `rounded border px-2 py-1 text-sm` (matching native select pattern from worker-form.tsx)
- Overall spacing: `space-y-4` between rows
- Responsive: rows stack vertically on mobile with appropriate spacing

IMPORTANT: Do NOT use Shadcn components. Use the project's custom Button and Card from `@/components/ui`. Use native `<select>` elements for time pickers.

IMPORTANT: All hidden inputs must have correct `name` attributes matching the server action's FormData parsing: `day_0_startTime`, `day_0_endTime`, `day_0_isAvailable` etc.
