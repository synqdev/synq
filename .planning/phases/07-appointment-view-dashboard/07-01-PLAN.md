  <name>Task 1: AdminSettings Prisma model and API endpoints</name>
  <files>
    prisma/schema.prisma
    app/api/admin/appointment/[id]/route.ts
    app/api/admin/appointment/today/route.ts
    app/api/admin/settings/route.ts
  </files>
  <action>
1. Add AdminSettings model to prisma/schema.prisma at the end (after Chat models):
   ```
   model AdminSettings {
     id             String   @id @default("default")
     aiProvider     String   @default("openai")
     businessType   String   @default("general")
     autoTranscribe Boolean  @default(true)
     recordingLang  String   @default("ja")
     audioQuality   String   @default("standard")
     updatedAt      DateTime @updatedAt
   }
   ```
   Run `npx prisma db push` to sync schema.

2. Create GET /api/admin/appointment/[id]/route.ts:
   - Auth guard with getAdminSession() (return 401 if not admin)
   - Fetch booking by ID with includes: customer (id, name, email, phone, locale, notes), worker (id, name, nameEn), service (id, name, nameEn, duration), karuteRecords (id, status, createdAt, ordered desc), recordingSessions (id, status, startedAt, ordered desc)
   - Return 404 if not found
   - Follow existing API route patterns (see app/api/admin/calendar/route.ts)

3. Create GET /api/admin/appointment/today/route.ts:
   - Auth guard with getAdminSession()
   - Accept optional `date` query param (defaults to today in JST via toZonedTime)
   - Fetch bookings for that day: startsAt >= startOfDay AND startsAt < endOfDay, status != CANCELLED
   - Include: customer (name), service (name, nameEn), worker (name), karuteRecords (id, status, take 1)
   - Order by startsAt asc
   - Return { bookings: [...] } with each booking including a computed `hasKarute` boolean

4. Create GET + PUT /api/admin/settings/route.ts:
   - GET: Auth guard, fetch AdminSettings with id="default". If not found, create default row and return it (upsert pattern).
   - PUT: Auth guard, parse body with Zod validation (aiProvider: string, businessType: string, autoTranscribe: boolean, recordingLang: string, audioQuality: string). Upsert to AdminSettings with id="default". Return updated settings.
   - Use zod for request body validation on PUT.
  </action>
  <verify>
    `npx prisma db push` succeeds without errors.
    `curl -b admin_session=... http://localhost:3000/api/admin/appointment/today` returns JSON with bookings array.
    `curl -b admin_session=... http://localhost:3000/api/admin/settings` returns default settings JSON.
  </verify>
  <done>
    AdminSettings model exists in schema. All three API routes return correct data with auth protection. Settings GET returns defaults on first call, PUT updates and returns new values.
  </done>
