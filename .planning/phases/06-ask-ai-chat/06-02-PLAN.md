All components are `'use client'` since they involve interactivity. Use Tailwind CSS for all styling. Use `useTranslations('admin.chat')` for i18n labels.

**1. `useChatStream.ts`** - Custom hook for consuming SSE stream:

```typescript
// Returns { sendMessage, isStreaming, streamingContent, error }
// Uses fetch + ReadableStream.getReader() for POST requests (NOT EventSource which is GET-only)
// Implements AbortController for cleanup on unmount
// Parses SSE events: { content }, { usage }, { conversationId }, [DONE]
// Accumulates content chunks into streamingContent state
// Disables sending while already streaming (prevents race conditions per RESEARCH pitfall 3)
// On [DONE], calls onComplete callback with full response text + conversationId
```

Key implementation details:
- Accept `onComplete(content: string, conversationId: string)` callback
- Use `useRef` for AbortController to cancel on unmount
- Split received text on `\n\n`, filter empty, parse each `data: ...` line
- Handle `data: [DONE]` as stream end signal
- Set error state if fetch fails or stream errors

**2. `ChatBubble.tsx`** - Individual message bubble:

- Props: `{ role: 'user' | 'assistant', content: string, citations?: { karuteId: string, label: string }[], createdAt: string }`
- User messages: Right-aligned, primary color background (bg-blue-600), white text
- AI messages: Left-aligned, light gray background (bg-gray-100), dark text
- Parse `[KR:uuid]` markers in content and replace with numbered superscript links `[1]` that link to `/admin/karute/{id}`
- Timestamps: Small gray text below each message
- Use `useLocale()` from next-intl for date formatting

**3. `ChatMessages.tsx`** - Message list container:

- Props: `{ messages: Message[], streamingContent: string | null }`
- Auto-scroll to bottom when new messages arrive or streaming content updates
- Use `useRef` on container + `useEffect` with scrollIntoView
- If `streamingContent` is non-null, render an extra assistant ChatBubble with the streaming content (no timestamp yet)
- Show empty state when no messages: centered text "Ask a question about this customer" / "Ask a question" (i18n)

**4. `ChatInput.tsx`** - Text input with send button:

- Single-line text input (not textarea) per user decision
- Send button with arrow/send icon
- Enter key to send (per user decision)
- Disabled state when `isStreaming` is true (prevents race conditions)
- Props: `{ onSend: (message: string) => void, isStreaming: boolean }`
- Clear input after sending
- Use the existing `Input` component from `@/components/ui/input` if it fits, otherwise create a clean custom input

**5. `QuickActions.tsx`** - Context-aware quick action buttons:

- Props: `{ customerId: string | null, onAction: (message: string) => void, isStreaming: boolean }`
- Customer-scoped mode (customerId present):
  - "前回の施術まとめ" / "Summarize last visit"
  - "施術履歴を表示" / "Show treatment history"
  - "アレルギー情報" / "Any allergies?"
  - "次回の予約" / "Next appointment"
- Global mode (customerId null):
  - "最近の施術傾向" / "Recent treatment trends"
  - "本日の予約一覧" / "Today's appointments"
- Small pill-shaped buttons in a horizontal scrollable row
- Each button calls `onAction` with the Japanese text of the query (AI responds in user locale regardless)
- Hidden when `isStreaming` is true

**6. `ChatProvider.tsx`** - React context for chat state:

- Provides: `{ isOpen, setIsOpen, customerId, setCustomerId, messages, setMessages, conversationId, setConversationId, refreshHistory }`
- Uses SWR to load chat history when `customerId` changes: `useSWR(customerId ? /api/admin/chat/history/${customerId} : null, fetcher)`
- When SWR data loads, update messages and conversationId state
- `refreshHistory` function triggers SWR mutate to reload
- Export `useChatContext` hook for consuming components

**7. `ChatPanel.tsx`** - Slide-over panel container:

- Renders as a fixed-position panel on the right side of the screen (per user decision: like Intercom)
- Uses Tailwind translate-x + transition for slide animation (per RESEARCH: don't hand-roll)
- Width: `w-96` on desktop, full-width on mobile
- Header: "AI Chat" / "AIチャット" title + close button (X icon)
- If `customerId` is set, show customer name in header subtitle
- Body: `ChatMessages` component
- Footer: `QuickActions` + `ChatInput`
- Backdrop overlay (semi-transparent) that closes panel on click
- Toggle button: Fixed floating action button (bottom-right corner) with chat icon to open the panel
- Z-index high enough to float above all admin content

**8. `index.ts`** - Barrel export:

Export `ChatPanel`, `ChatProvider`, `useChatContext` for use in admin layout.
