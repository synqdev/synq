**Extend `src/lib/services/customer.service.ts`** with:

1. `getCustomerDetail(id: string)` - Returns customer with:
   - All customer fields
   - Bookings (last 50, ordered by startsAt desc) including service name, worker name, resource name, status, price
   - Medical records (ordered by enteredAt desc) with item title and content type
   - Computed: visitCount (CONFIRMED bookings count), lastVisitDate, nextBookingDate (future CONFIRMED booking)
   - Assigned staff name (if assignedStaffId set, join to Worker)

2. `updateCustomerNotes(id: string, notes: string)` - Updates the `notes` field on Customer. Returns updated customer.

3. `updateCustomerAssignedStaff(id: string, assignedStaffId: string | null)` - Updates assignedStaffId.

**Create `app/api/admin/customers/[id]/route.ts`**:

GET handler:
1. Verify admin session
2. Extract `id` from params
3. Call `getCustomerDetail(id)`
4. Return 404 if not found
5. Return customer detail JSON

**Create `app/api/admin/customers/[id]/notes/route.ts`**:

PUT handler:
1. Verify admin session
2. Extract `id` from params
3. Parse body: `{ notes: string, assignedStaffId?: string | null }`
4. Validate with Zod (notes max 5000 chars)
5. Call `updateCustomerNotes` and/or `updateCustomerAssignedStaff`
6. Return updated customer
  </action>

- Invalid ID returns 404
