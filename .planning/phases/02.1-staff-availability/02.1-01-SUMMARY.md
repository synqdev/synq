- `messages/en.json` - Added admin.schedulePage with English UI strings

## Decisions Made

- **findFirst+update/create pattern**: WorkerSchedule has no unique constraint on (workerId, dayOfWeek), so standard Prisma upsert cannot be used. The findFirst+update/create pattern handles this correctly.
- **ZodError handling**: Caught and returned as `{ success: false, error: 'Validation failed' }` for UI display; other unexpected errors are rethrown so they surface as server errors.
- **isAvailable=false skips time validation**: When a worker is marked unavailable for a day, the start/end times are irrelevant, so the Zod refine only applies when `isAvailable: true`.
