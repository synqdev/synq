- Register Noto Sans JP font for Japanese text rendering:
  ```

    family: 'NotoSansJP',

      { src: 'https://cdn.jsdelivr.net/fontsource/fonts/noto-sans-jp@latest/japanese-400-normal.ttf' },
      { src: 'https://cdn.jsdelivr.net/fontsource/fonts/noto-sans-jp@latest/japanese-700-normal.ttf', fontWeight: 'bold' },
    ],
  });
  ```
  Note: If CDN URLs don't work at runtime, fall back to bundling font files in `public/fonts/`. Test during implementation.
- StyleSheet with: page padding 40, fontFamily 'NotoSansJP', fontSize 10, section margins, category headers bold
- Props: `{ record: KaruteRecordWithEntries }` containing record data with entries, customer, worker
- Document structure:
  - Title: "施術記録 / Treatment Record"
  - Header section: date (formatted), practitioner name, customer name, status
  - Summary section: record.summary text
  - Entries grouped by category:
    - Category header (Japanese label, e.g., "症状", "施術", etc.)
    - Each entry as a bullet: `- {content}`
  - Footer: generation timestamp
- Helper: `categoryLabel(category: string)` mapping enum values to Japanese labels
- Helper: `groupBy(entries, 'category')` to group entries by category

**3. Create `src/lib/services/karute-export.service.ts`:**
- Import renderToBuffer from '@react-pdf/renderer'
- Import KaruteDocument from '@/components/karute/KaruteDocument'
- Import prisma for data fetching

Export `async function generateKarutePDF(recordId: string): Promise<{ success: true; buffer: Buffer } | { success: false; error: string }>`:
- Fetch record with entries (ordered by displayOrder), customer, worker
- Return error if not found
- Call `renderToBuffer(<KaruteDocument record={record} />)`
- Return buffer wrapped in success result

Export `async function generateKaruteText(recordId: string): Promise<{ success: true; text: string } | { success: false; error: string }>`:
- Fetch same data as PDF
- Build formatted text string:
  ```
  施術記録 / Treatment Record
  ──────────────────────────
  日付: {date}
  施術者: {worker.name}
  患者: {customer.name}
  ステータス: {status}

  【サマリー】
  {summary}

  【症状】
  - {entry.content}
  ...

  【施術】
  - {entry.content}
  ...
  ```
- Group entries by category, output each section
- Return text wrapped in success result

**4. Create `app/api/admin/karute/[id]/export/route.ts`:**
- Export GET handler with admin auth check
- Read `format` query param: 'pdf' (default) or 'text'
- If pdf: call generateKarutePDF, return Response with content-type 'application/pdf', content-disposition 'attachment; filename=karute-{id}.pdf'
- If text: call generateKaruteText, return Response with content-type 'text/plain; charset=utf-8', content-disposition 'attachment; filename=karute-{id}.txt'
- Return 500 on error
