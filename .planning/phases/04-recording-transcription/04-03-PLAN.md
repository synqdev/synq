  - Gets time-domain data via `analyserNode.getByteTimeDomainData()`.
  - Draws waveform on canvas: dark background (`rgb(30, 41, 59)` — slate-800), waveform line in a teal/cyan color (`rgb(94, 234, 212)` — teal-300) to match a medical/clinical feel.

- Stores `requestAnimationFrame` ID in a ref, cancels with `cancelAnimationFrame` on cleanup and when isActive becomes false.

**2. `src/components/recording/RecordingPanel.tsx`:**

Props:
```typescript
{
  customerId: string;
  workerId: string;
  karuteRecordId?: string;
  bookingId?: string;
}
```

This is the main orchestrating component. It composes all sub-components and manages the recording pipeline.

**State management:**
- Uses `useAudioRecorder()` hook for recording state.
- Local state for: `sessionId` (string | null), `isUploading` (boolean), `isTranscribing` (boolean), `transcriptionSegments` (array), `pipelineError` (string | null).

**Pipeline flow (wired in event handlers):**

a. **handleStart:**
   - Call `createRecordingSessionAction({ customerId, workerId, karuteRecordId, bookingId })` to create RecordingSession in DB.
   - Store returned `sessionId`.
   - Call `recorder.startRecording()`.

b. **handleStop:**
   - Call `recorder.stopRecording()` to get Blob.
   - Set `isUploading = true`.
   - Upload Blob via `fetch('/api/admin/recordings/upload', { method: 'POST', body: formData })` where formData has `file` (Blob named `{sessionId}.webm`) and `recordingSessionId`.
   - Set `isUploading = false`, `isTranscribing = true`.
   - Trigger transcription via `fetch('/api/admin/recordings/transcribe', { method: 'POST', body: JSON.stringify({ recordingSessionId: sessionId }) })`.
   - On transcription success, fetch the recording session with segments via `getRecordingSession` (or a new fetch to get segments).
   - Set `transcriptionSegments` from response.
   - Set `isTranscribing = false`.
   - On any error in the pipeline, set `pipelineError` with descriptive message.

**Layout:**
- Card-style container with padding.
- Top: RecordingTimer (centered, prominent).
- Middle: WaveformVisualizer (full width).
- Below waveform: RecordingControls.
- Status indicators: show "Uploading..." or "Transcribing..." with Spinner during those phases.
- Error display: show pipelineError in red text if set.
- Bottom: TranscriptionDisplay (shown after transcription completes, with segments).
- Use `useTranslations('admin.recording')` for status messages.
