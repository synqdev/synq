<task type="auto">
  <name>Task 1: Extend Prisma schema with Phase 2 models and fields</name>
  <files>prisma/schema.prisma</files>
  <action>
Add the following to the existing Prisma schema. All changes are ADDITIVE -- do not modify or remove any existing models or fields.

**New models to add (after existing models):**

1. `TreatmentCategory` - id (cuid), name (String), displayOrder (Int, default 0), isActive (Boolean, default true), createdAt, updatedAt. Has relation to Service[].

2. `TreatmentOption` - id (cuid), name (String), price (Int), displayOrder (Int, default 0), isActive (Boolean, default true), createdAt, updatedAt.

3. `MedicalRecordItem` - id (cuid), title (String), contentType (String, values 'text'|'image'), isPublic (Boolean, default false), displayOrder (Int, default 0), isActive (Boolean, default true), createdAt, updatedAt. Has relation to MedicalRecord[].

4. `MedicalRecord` - id (cuid), customerId (String), itemId (String), content (String? @db.Text), imageUrl (String?), enteredBy (String), enteredAt (DateTime, default now()), updatedAt. Relations to Customer and MedicalRecordItem. Index on [customerId].

5. `Shift` - id (cuid), workerId (String), date (DateTime @db.Date), isScheduled (Boolean, default true), createdAt, updatedAt. Relation to Worker. @@unique([workerId, date]), @@index([date]).

6. `CashRegisterClose` - id (cuid), date (DateTime @db.Date), bills10000/bills5000/bills1000 (Int, default 0), coins500/coins100/coins50/coins10/coins5/coins1 (Int, default 0), totalCash (Int), cardPayments (Int, default 0), otherPayments (Int, default 0), expectedTotal (Int), variance (Int), closedBy (String), closedAt (DateTime, default now()). @@unique([date]).

7. `EmailHistory` - id (cuid), customerId (String), recipientEmail (String), subject (String), body (String @db.Text), status (String, values 'sent'|'failed'|'bounced'), sentAt (DateTime). Relations to Customer. @@index([customerId]), @@index([sentAt]).

**Extensions to existing models:**

1. `Customer` - Add fields: `assignedStaffId String?`, `visitCount Int @default(0)`, `lastVisitDate DateTime?`, `outstandingAmount Int @default(0)`. Add relations: `medicalRecords MedicalRecord[]`, `emailHistory EmailHistory[]`.

2. `Service` - Add fields: `categoryId String?`, `enableOnlineBooking Boolean @default(true)`, `imageUrl String?`, `displayOrder Int @default(0)`. Add relation: `category TreatmentCategory? @relation(fields: [categoryId], references: [id])`.

3. `Booking` - Add fields: `ticketCount Int?`, `consumedAmount Int?`, `notes String? @db.Text`.

4. `Worker` - Add relation: `shifts Shift[]`.

**Important:** Do NOT add a storeId to CashRegisterClose. The current schema is single-store. Keep it simple.
